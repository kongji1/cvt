    常见问题

    格式转换遵循“能证明才生成”的原则：保留可验证字段，明确标注不可逆边界，不补造凭证。

    凭证会上传到服务器吗？

    普通解析、识别与格式转换均在浏览器本地完成。只有 Agent Identity 的 Runtime 注册步骤需要显式点击“注册并生成”后才会发起网络请求，私钥仍只在本地生成。

    为什么有些格式不能反向转换？

    API Key、缺失 refresh_token 的会话、Agent Identity 私钥等字段不能从另一种容器格式凭空恢复。页面只展示当前输入实际可安全生成的目标格式。

    历史记录是否安全？

    历史记录默认关闭。开启前会再次确认；开启后原始输入只保存在当前浏览器的 localStorage，最多保留最近 6 条，你可以随时关闭并清空。

    支持哪些文件和容器？

    支持 JSON、JSONL / NDJSON、连续 JSON、部分 TXT 导出和多文件合并。输出可能是单个 JSON、账号拆分 ZIP 或 CPA 原生 auth ZIP，具体取决于输入和目标格式。
